Priority Gaps to Fix

🔴 Critical — Blocking Core Stage 2 Functionality

  • AdvancedPropertyEditor not wired into the live page editor — admin/pages/[id]/page.tsx still uses the basic PropertyEditor. ComponentTreePanel is also disconnected. Users cannot access the advanced 6-tab editor at all.
  • No themes table / Theme entity — The full theme preset save/load/activate system (as specified in task 1.1 and 4.1) is absent. Only flat key-value theme settings exist.
  • No Staging backend — StagingController and StagingSyncService are completely missing. The staging page is 100% mock.
  • No Icon Library backend API — GET /api/admin/icons and GET /api/admin/icons/:pack not implemented.
  • No side-by-side diff viewer — Version history rollback exists in the API but zero UI to see/compare versions.

🟡 Medium — Significant Feature Gaps

  • Pattern background missing in BackgroundTab (SVG dot/grid/stripe patterns).
  • No media library picker in the property editor — image/video/file fields require manual URL entry.
  • No custom SVG upload in IconPickerModal; only Lucide icons (~250) vs. required ~10k+.
  • Bulk edit modal not implemented (BulkActions only does delete/approve/reject/publish).
  • Bulk duplicate + archive modals missing.
  • BulkActions not integrated into actual content list pages.
  • Dynamic data binding not rendered — dataSource values stored but PageSectionRenderer does not fetch/inject CMS data at runtime.
  • inlineCss and responsive breakpoint overrides not applied at render time in PageSectionRenderer.
  • No workflow → email hook — approval requests don't trigger actual email sends.
  • 50+ system preset seed data missing — no SQL migration seeds preset records into component_presets.

🟡 Low — Minor / Polish Gaps

  • Text-transform, text-decoration, text-shadow missing from StyleTab typography controls.
  • Per-corner border radius not implemented (only single radius value).
  • Color picker: no hex/RGB/HSL toggle, no saved theme color swatches.
  • Gradient editor: no interactive stop editor (drag stops).
  • Staging toggle missing from admin header/sidebar.
  • Stage 2 services (ComponentPresetService, ContentVersionHistoryService, WorkflowService) have zero unit tests.
  • Admin/themes page missing border/shadow token groups.
  • RichTextEditor not integrated into PropertyEditor content tab.
  • No Cypress E2E or frontend component tests.
