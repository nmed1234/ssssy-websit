Step 1 — Deploy Frontend to Vercel FREE
  1. Go to vercel.com → Sign up with GitHub
  2. Click "Add New Project" → Import your GitHub repo
  3. Set Root Directory to frontend
  4. Add environment variable: NEXT_PUBLIC_API_URL = your backend URL
  5. Click Deploy — done in ~2 minutes
  6. Optional: Add a custom domain (free on Vercel)
Make sure your repo is on GitHub/GitLab first. Vercel auto-deploys on every push.
Step 2 — Create PostgreSQL Database on Neon.tech FREE FOREVER
  1. Go to neon.tech → Sign up (free)
  2. Create a new project → name it ssssy_db
  3. Copy the Connection String — looks like:
    postgresql://user:password@ep-xxx.neon.tech/ssssy_db?sslmode=require
  4. Run your SQL migrations using Neon's SQL Editor or a local psql connection
Step 3 — Deploy Spring Boot Backend to Railway $5 FREE CREDIT/MO
  1. Go to railway.app → Sign up with GitHub
  2. Click "New Project" → Deploy from GitHub repo
  3. Set Root Directory to backend
  4. Railway detects the Dockerfile automatically
  5. Add these environment variables:
VariableValue
SPRING_DATASOURCE_URLYour Neon connection string
SPRING_DATASOURCE_USERNAMENeon DB username
SPRING_DATASOURCE_PASSWORDNeon DB password
JWT_SECRETAny 64-char random string
SERVER_PORT8080
  1. Click Deploy — Railway builds the Docker image and starts the app
  2. Copy the generated Railway URL (e.g. https://ssssy-backend.up.railway.app)
  3. Go back to Vercel and update NEXT_PUBLIC_API_URL to this URL
Alternative: Use Render.com (free 750 hrs/mo) but the service sleeps after 15 min of inactivity — first request after sleep takes ~30 sec.

Important: CORS Configuration

After deploying, you must update your backend CORS config to allow your Vercel domain. Find the CORS configuration in your Spring Boot app and add your Vercel URL:


allowedOrigins: https://your-app.vercel.app
Without this, the frontend browser calls to the backend API will be blocked.
